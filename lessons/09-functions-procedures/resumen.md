# Resumen: Functions y Procedures

## Lo esencial

- `function` devuelve un valor o filas
- `procedure` ejecuta acciones
- `SELECT` llama funciones
- `CALL` ejecuta procedimientos
- `RETURNS TABLE` y `RETURNS SETOF` sirven para devolver conjuntos

## Lo que debe salirte natural

- crear una funcion simple
- devolver filas desde una function
- ejecutar una procedure que cambia datos
- decidir si la logica merece vivir en la base

## Siguiente paso

Cuando esto ya salga bien, pasamos a:
- `VIEW`
- `MATERIALIZED VIEW`
- `TRIGGER`
