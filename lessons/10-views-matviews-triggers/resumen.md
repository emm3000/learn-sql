# Resumen: Views, Materialized Views y Triggers

## Lo esencial

- `VIEW` reutiliza consultas
- `MATERIALIZED VIEW` guarda resultados calculados
- `TRIGGER` automatiza acciones al escribir datos

## Lo que debe salirte natural

- crear una view para simplificar lectura
- elegir una materialized view para reportes frecuentes
- usar un trigger para automatizar un campo como `updated_at`

## Siguiente paso

Cuando esto ya salga bien, pasamos a:
- transacciones
- locks
- concurrencia
