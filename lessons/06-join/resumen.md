# Resumen: JOIN

## Lo esencial

- `JOIN` combina datos de tablas relacionadas.
- `INNER JOIN` devuelve coincidencias.
- `LEFT JOIN` conserva la tabla izquierda completa.
- El `ON` define la relacion entre tablas.

## Lo que debe salirte natural

- unir empleados con departamentos
- unir clientes con pedidos
- mostrar datos principales con datos relacionados
- usar alias de tablas para leer mejor

## Siguiente paso

Cuando esto ya salga bien, pasamos a:

- `GROUP BY`

- agregaciones
